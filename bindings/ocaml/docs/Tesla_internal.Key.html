Module Tesla_internal.Key

module Key: sig .. end
A TESLA instance can be identified by a tesla_class and a tesla_key. This key represents the values of event parameters (e.g. a credential passed to a security check), some of which may not be specified.

Clients can use tesla_key to look up sets of automata instances, using the bitmask to specify don't-care parameters.

Keys can hold arbitrary integers/pointers.
