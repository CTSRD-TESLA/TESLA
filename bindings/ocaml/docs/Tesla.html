type ctx =
| Global (*The single global context*)
| Thread (*A per-thread context*)
A context where TESLA data is stored.

TESLA data can be stored in a number of places that imply different synchronisation requirements. For instance, thread-local storage does not require synchronisation on access, whereas global storage does. On the other hand, thread-local storage cannot be used to track events across multiple threads.

module Class: sig .. end
A description of a TESLA automaton, which may be instantiated a number of times with different names and current states. 
val with_class : store:Store.t ->
       id:Class.id ->
       name:string -> descr:string -> f:(Class.t -> 'a) -> 'a
Retrieve a Tesla.Class.t with the given id and apply f on it. The class will always be released when f terminates, even if f raises an exception. However, f should not attempt to retrieve the same class name in the same thread or global context, or the function will deadlock.
